# PR body template

Use and adapt this structure:

```md
## Context

This PR addresses #<issue-number>.

[Brief explanation of the issue and the interpretation taken if needed.]

## What changed

- change 1
- change 2
- change 3

## Notes

- assumptions / limitations / follow-up notes if relevant
```

## Issue reference guidance

- Use `Closes #<n>` when you are confident the PR fully resolves the issue.
- Use `Refs #<n>` when the PR is partial, exploratory, or based on an interpretation that might need confirmation.

## Style notes

- Keep it compact.
- Explain intent, not every tiny edit.
- Mention ambiguity when you had to choose an interpretation.
- Call out validation done when it matters.
