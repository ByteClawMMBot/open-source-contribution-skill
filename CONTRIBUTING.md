# Contributing

Thanks for contributing.

## Before you start

Please check the repository materials first:

- `README.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- existing issues and pull requests

## Preferred workflow

Use a normal contributor flow:

1. Fork the repository
2. Create a focused branch
3. Keep the change scoped to one issue or one small improvement
4. Write a clear commit message
5. Open a pull request with context and a concise summary

## Branch naming

Examples:

- `issue-12-fix-readme-links`
- `docs/issue-3-improve-contributing-guide`
- `feat/issue-9-add-export-command`

Avoid baking your own username into branch naming unless the target project explicitly wants that.

## Pull requests

A good PR should include:

- context
- what changed
- issue reference
- assumptions or limitations if relevant

Try to avoid unrelated cleanup in the same PR.

## Scope discipline

This repo is small on purpose. Prefer crisp, reviewable changes over broad refactors.

## Issues

Use the issue templates when possible. If you are proposing a new direction, explain the problem first, then the proposed solution.
